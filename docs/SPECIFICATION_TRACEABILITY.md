# GRAFT-GS specification-to-code traceability

Audit date: 2026-07-16. The mathematical objective in `../../GRAFT-GS.md` is
mapped here to the current executable repository. A status describes both the
implementation and the strongest validation actually available; compilation is
not treated as numerical validation.

Status vocabulary: **faithful and validated**, **strengthened beyond the
specification**, **implemented but unvalidated**, **approximate**, **partially
integrated**, **scientifically weak**, **incorrect**, **missing**, and
**externally blocked**.

## Actual production path

`MeshFleetObjectDataset` -> audited RGB/alpha/cameras/surface contract ->
`VGGTAdapter` -> differentiable scene-level Sim(3) alignment ->
`GeometricEvidenceBuilder` -> optional same-object autograd all-gather ->
`PersistentOctreeAtlas.from_evidence` -> TRELLIS hidden support seeding ->
implicit sparse unbalanced OT -> differentiable adaptive chart refit ->
OT/uncertainty-biased GSTA -> finite topology proposal and hard admissibility ->
product-manifold state -> CBF-projected safe Heun flow -> analytical surface
Gaussians and the same stratum mesh -> renderer and deterministic PLY/GLB.

Training uses the same `GraftGS.forward` path with explicit stage stops:
Phase A `evidence_calibration`, Phase B `atlas_autoencoding`, Phase C
`flow_pretraining`, and Phases D-F `full`. Inference uses `full` only.

## External validation and optimized-backend boundaries

The following are explicitly mapped as external validation or optimization
boundaries rather than silently claimed: validation of the optional pinned
learned perceptual checkpoint, true matched-precision low-bit backward
kernels, and fused sparse CUDA/Triton kernels. They are prioritized
in `UNRESOLVED_BLOCKERS.md` according to whether they are implementable locally
or require checkpoints/A800 execution.
